Sources and patch for implement IPRT based swagger code generators.

To integrate this ($THIS) into a swagger-codegen source tree ($TREE):
  ln -s $THIS/java/io/swagger/codegen/languages/IprtClientCodegen.java \
        $TREE/./modules/swagger-codegen/src/main/java/io/swagger/codegen/languages/IprtClientCodegen.java
  ln -s $THIS/resources/iprt-client/ \
        $TREE/modules/swagger-codegen/src/main/resources/iprt-client
  ln -s $THIS/bin/iprt-client-petstore.sh \
        $TREE/bin/iprt-client-petstore.sh
  ln -s $THIS/bin/windows/iprt-client-petstore.bat \
        $TREE/bin/windows/iprt-client-petstore.bat
  cd $TREE && patch -p1 -i $THIS/patch.diff

To build swagger (JDK 8, not 10!) and generate the petstore project:
  mvn -T 2C -o -DskipTests -Dmaven.javadoc.skip=true -pl modules/swagger-codegen-cli -am package \
    && bash ./bin/iprt-client-petstore.sh

If you only work on the mustache templates, you can save yourself a lot of
waiting by pointg the swagger-codegen at resources/iprt-client.  For instance:
  java -jar swagger-codegen-cli.jar generate -l iprt -i spec.yml -o outdir \
        --additional-properties templateDir=/mnt/.../resources/iprt-client/

P.S. From behind a proxy you need to edit mavend/conf/settings.xml to set up
     the proxy (http and https share config).  And because of some bits being
     downloaded by a badly integrated plugin, you need to add the following
     defines to the mvn command lines:
        -Dhttp.proxyHost=<my.proxy.com>  -Dhttp.proxyPort=<my-proxy-port> \
        -Dhttps.proxyHost=<my.proxy.com> -Dhttps.proxyPort=<my-proxy-port>
